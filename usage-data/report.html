Buggy Code and Tool Failures
You frequently encounter scripts that fail on first execution due to coding errors—wrong function usage, duplicate blocks, leftover statements, or incorrect parameters—requiring debugging passes before the task completes. You'd benefit from asking Claude to run and verify each script immediately after writing it, rather than discovering errors at the next workflow step.

  • A curl command used -d (URL-encoding the JS payload and breaking syntax) instead of --data-binary, causing the initial script to fail and forcing a re-scrape to save data that was only held in memory.
Web Scraping and Browser Automation Instability
You rely heavily on CDP browser automation and web scraping workflows that are fragile—premature tab closures cause data loss, extraction methods fail on dynamic content requiring fallback approaches, and agent stalling or network errors interrupt mid-task. You could reduce this friction by building explicit retry/fallback logic and data persistence checkpoints into your scraping skills from the start.
  • Browser tabs were closed prematurely during a scraping task, causing complete data loss and requiring a full re-scrape of the dataset.
  • Course 3 generation agent stalled 3 consecutive times hitting the 600s watchdog timeout, forcing a fallback to manual file writing by the main agent and losing the parallel agent workflow you wanted.
Inaccurate Initial Results Requiring User Correction
You often receive results that are close but wrong in specific ways—wrong links, wrong directory names, missed items, or misdiagnosed issues—requiring you to manually spot and report the error before Claude corrects it. You could mitigate this by prompting Claude to verify assumptions against actual file paths and URLs before presenting results.
  • The skill-vetter link was wrong twice in a row, requiring you to catch and report the error both times before the correct link was used.
  • Claude misidentified your input method issue as Chinese-English auto-spacing when the actual problem was full-width character mode; you had to provide a visual example to redirect the diagnosis, wasting time on the wrong fix path.
