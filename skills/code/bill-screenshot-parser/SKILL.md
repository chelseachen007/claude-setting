# 账单截图解析器

解析支付宝和微信账单截图，提取交易信息并导出为 Excel 文件。

## 工作流程

### 1. 接收截图
用户上传一张或多张支付宝/微信账单截图。

### 2. 解析截图
使用 Claude 图像识别能力解析截图，提取以下信息：

**支付宝字段：**
- 交易时间
- 交易类型（支出/收入）
- 金额
- 商品/商户名称
- 订单号
- 支付方式

**微信字段：**
- 交易时间
- 交易类型（支出/收入）
- 金额
- 商品/商户名称
- 交易单号
- 支付方式

### 3. 数据标准化
将提取的数据统一为以下格式：

```json
{
  "date": "2026-02-27 12:30:00",
  "type": "expense",
  "amount": 25.50,
  "category": "餐饮",
  "description": "美团外卖-黄焖鸡米饭",
  "platform": "alipay",
  "orderId": "2026022722001401234567890"
}
```

### 4. 导出 Excel
调用 `scripts/export_to_excel.py` 生成 Excel 文件。

## 字段映射规则

| 原始字段 | 标准字段 | 说明 |
|---------|---------|------|
| 交易时间/付款时间 | date | 统一格式 YYYY-MM-DD HH:mm:ss |
| 支出/收入 | type | expense/income |
| 金额/订单金额 | amount | 数值，保留两位小数 |
| 商品/商户/交易说明 | description | 交易描述 |
| 支付宝/微信 | platform | alipay/wechat |
| 订单号/交易单号 | orderId | 原始订单号 |

## 类别推断规则

根据商品/商户名称自动推断类别：

| 类别 | 关键词 |
|-----|-------|
| 餐饮 | 外卖、餐厅、咖啡、奶茶、超市、美团、饿了么、肯德基、麦当劳 |
| 交通 | 打车、滴滴、地铁、公交、加油、停车、高德 |
| 购物 | 淘宝、京东、拼多多、商场、天猫、得物 |
| 娱乐 | 电影、游戏、音乐、视频、爱奇艺、腾讯视频、B站 |
| 医疗 | 医院、药店、诊所、体检 |
| 教育 | 课程、书籍、培训、知乎 |
| 转账 | 转账、红包、AA收款 |
| 生活 | 水费、电费、燃气、话费、宽带 |
| 其他 | 无法归类的交易 |

## 过滤规则

导出时**默认排除**以下类型的交易：

| 排除类型 | 关键词 |
|---------|-------|
| 理财收益 | 余额宝、收益发放、基金分红、理财 |
| 系统消息 | 您于、使用"先、商户单号 |

如用户需要保留这些记录，可指定 `--no-filter` 参数。

## 脚本使用

```bash
# 生成新的 Excel（默认过滤理财收益）
python3 scripts/export_to_excel.py --data '[{...}]' --output ./账单.xlsx

# 追加到已有 Excel
python3 scripts/export_to_excel.py --data '[{...}]' --output ./账单.xlsx --append

# 保留所有记录（不过滤）
python3 scripts/export_to_excel.py --data '[{...}]' --output ./账单.xlsx --no-filter
```

## 注意事项

1. **金额识别**：注意区分「-」表示支出，「+」表示收入
2. **日期格式**：不同截图格式可能不同，需统一为 YYYY-MM-DD HH:mm:ss
3. **平台识别**：支付宝截图通常有蓝色主题，微信有绿色主题
4. **隐私信息**：订单号等敏感信息可选择脱敏处理
5. **多截图合并**：批量处理时自动合并为单个 Excel 文件
6. **重复检测**：追加时根据订单号检测重复记录
