# Article Extractor - 个人风格文章摘录

按照用户的个人笔记风格从文章或视频中提取关键信息并创建结构化笔记。

## 工作流程

### 第1步:获取内容

**网页文章**:使用 `mcp__web_reader__webReader` 获取 markdown 格式内容
**视频**:使用 `video-downloader` skill 下载并提取字幕/转录文本

### 第2步:快速评估

分析内容类型:
- 概念解释类(单一概念,简单明了)
- 方法论/框架类(有完整的理论框架)
- 技术类(代码、架构、最佳实践)
- 观点评论类(个人观点和评论)
- 书籍类(完整书籍的读书笔记)

确定核心信息和价值点。

### 第3步:选择模板并创建笔记

根据内容类型使用对应模板(见 `references/templates.md`)

**关键原则**:
- 简单概念:极简记录(1-3句话)
- 复杂主题:详细展开(结构化列表)
- 技术主题:结构化拆解(核心概念、特性、优缺点)
- 记录"怎么做"而不只是"是什么"
- 用 `#细节` 标注重要数据/研究发现

### 第4步:知识链接和深度思考

**核心**:这是用户笔记风格的关键特征,必须特别重视。

#### 4.1 搜索已有知识连接

使用 Glob 工具搜索相关笔记:
- 按关键词搜索卡片目录
- 查找相同主题、相关概念、对立观点
- 查找可能的应用场景笔记

**示例**:如果文章讲"持续学习",应该搜索:
- `卡片/*学习*`
- `卡片/*Agent*`
- `卡片/*OpenClaw*`
- 可能找到: `[[量化思维]]`、`[[行动系统]]`、`[[Agent]]` 等

#### 4.2 建立知识链接

在"思考"部分建立连接:
- 直接相关:用 `[[卡片名]]` 连接
- 主题关联:用 `[[索引/主题索引]]` 连接
- 对比观点:指出与已有笔记的相同/不同之处

**用户真实示例**:
- `[[信息经纪人]]` 引用 `[[好奇心]]`、`[[AI]]`
- `[[认知的三种模式]]` 与 OpenClaw 联系

#### 4.3 产生新思考

**不只是连接,还要产生新的洞察**:

1. **补充视角**:这个知识如何补充或修正已有认知?
2. **应用迁移**:这个理论可以应用到我正在做的某事吗?
3. **趋势判断**:这个信息暗示了什么趋势或变化?
4. **行动启发**:我应该立即采取什么行动?
5. **知识缺口**:还缺什么信息才能完整理解?

**示例思考框架**:
```markdown
## 思考

**与已有知识的联系**:
- 这补充了 [[相关笔记1]] 中的XX概念
- 与 [[相关笔记2]] 的观点一致/互补/对立
- 这解释了为什么我在 [[相关笔记3]] 中遇到的XX现象

**新的洞察**:
- [这个知识改变了我的什么认知?]
- [这暗示了什么趋势或机会?]
- [我可以如何应用这个知识?]

**待探索的问题**:
- [还缺什么信息?]
- [需要进一步验证什么?]
```

#### 4.4 思考质量标准

**好的思考** (用户风格):
- ✅ 具体指出与哪些笔记相关
- ✅ 说明关联的逻辑(不只是列链接)
- ✅ 产生新的洞察或行动启发
- ✅ 提出待探索的问题

**不好的思考**:
- ❌ 只说"很有用"、"很重要"
- ❌ 没有建立具体的知识连接
- ❌ 没有产生新的思考或行动点

### 第5步:质量检查

1. 如果6个月后再看这条笔记,能快速理解核心内容吗?
2. 有没有建立与已有知识的连接?
3. 是否有可操作的行动点?
4. 标签是否准确?(便于后续搜索)
5. 出处是否完整?(便于追溯)

### 第6步:展示给用户确认

**重要**:在保存笔记前,必须向用户展示完整内容并获得批准。

展示格式:
```
📝 我已为文章/视频创建笔记,请查看:

---
[完整笔记内容]
---

是否保存到 卡片/?
- 确认:保存
- 修改:告诉我需要调整什么
- 取消:不保存
```

用户确认后才使用 Write 工具保存到 `卡片/` 目录。

## 内容风格特征

用户的笔记风格(基于 2159+ 篇笔记分析):

1. **结构化但不僵化**:根据内容复杂度灵活调整
2. **实用主义导向**:注重可操作性和方法论
3. **链接驱动**:大量使用 `[[wikilink]]` 建立知识网络
4. **简洁精炼**:避免冗余,保留核心信息
5. **个人视角**:不只是摘录,还有自己的思考和观点
6. **深度思考**:主动搜索相关笔记,建立连接,产生新洞察

## 笔记模板

详见 `references/templates.md`,包含:
- 极简卡片(概念类)
- 知识卡片(方法论/框架类)
- 技术卡片(技术类)
- 读书笔记(书籍类)

## 存放位置

- 原子化概念 → `卡片/`
- 文件命名:使用主题名称,如 `卡片/认知的三种模式.md`

## 标签策略

- 主题/领域标签(如 `AI/认知模式`)
- 类型标签(如 `智能理论`)
- 具体概念标签(如 `KevinKelly`)

## 语言偏好

- 中文为主
- 技术术语保留英文
- 列表优于段落
- 避免过度格式化
