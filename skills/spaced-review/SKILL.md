# 间隔重复复习 Skill

基于艾宾浩斯遗忘曲线的间隔重复复习系统，通过**主动回忆问答**帮助你巩固 Obsidian 知识卡片。

## 使用方式

```bash
# 开始今日复习
/review

# 或者说：
# "帮我复习今天的知识卡片"
# "开始间隔重复复习"
```

## 复习原理

基于笔记的 `date` 字段（创建日期），在以下时间节点自动提醒复习：

| 节点 | 时间 | 目的 |
|------|------|------|
| 第1次 | +1天 | 短期记忆巩固 |
| 第2次 | +3天 | 第一次复习 |
| 第3次 | +7天 | 周期复习 |
| 第4次 | +30天 | 月度复习 |
| 第5次 | +90天 | 季度复习 |

## 复习流程

1. **扫描**：找出今天需要复习的卡片
2. **问答**：根据卡片内容生成 2-3 个问题
3. **作答**：用户尝试回答（不看笔记）
4. **评分**：Claude 对回答进行评分
5. **反馈**：展示正确答案和笔记内容
6. **通过**：得分 ≥ 60% 才算通过该卡片

## 评分标准

| 分数 | 标准 |
|------|------|
| 100% | 完全正确，回答全面 |
| 80% | 基本正确，有小瑕疵 |
| 60% | 部分正确，有遗漏 |
| 40% | 只记得一点 |
| 20% | 完全忘记 |
| 0% | 答非所问 |

## 执行指令

当用户触发此 skill 时，按以下步骤执行：

### Step 1: 扫描卡片

```bash
python3 ~/.claude/skills/spaced-review/scripts/review.py
```

解析输出获取需要复习的卡片列表。

### Step 2: 逐个复习

对于每张卡片：

1. **检查卡片质量**：
   - 若卡片只有 frontmatter（无实际内容）→ 标记 ⚠️ 跳过，提示需补充
   - 若卡片只有链接（如 `[[其他笔记]]`）→ 标记 ⚠️ 跳过，建议合并或补充思考
   - 有效卡片 → 继续复习流程

2. **先不展示笔记内容**
3. 根据笔记内容生成 2-3 个关键问题
4. 等待用户回答
5. 对每个回答评分并给出反馈
6. 计算该卡片的总分
7. 展示完整笔记内容供对照
8. 分数 ≥ 60% 标记为通过
9. **低分卡片（<60%）**：提供知识扩展建议

### Step 3: 复习报告

完成后输出：
- 总卡片数
- 通过数量
- 未通过数量（需要重点复习）
- 每张卡片的得分

## 知识扩展

### 触发条件

| 情况 | 建议操作 |
|------|----------|
| 复习得分 < 60% | 提供扩展建议，询问是否补充卡片 |
| 用户要求"加深印象" | 提供案例/故事扩展 |
| 随机复习久远卡片 | 主动提供扩展内容 |

### 扩展框架（5层结构）

为低分卡片提供扩展时，按以下框架组织：

```markdown
## 核心论点
- 分层呈现：简单版 → 完整版 → 深入版
- 用 bullet points 而非长段落

## 案例故事
- 用生动场景解释抽象概念
- 可以是虚构故事或真实历史
- 关键：让读者能"看到画面"

## 现实应用
- 与真实世界/历史事件关联
- 回答"这为什么重要"

## 记忆钩子
- 一句话总结，最好用比喻
- 例如："科学验证像打包购买"

## 关联笔记
- [[相关概念1]]
- [[相关概念2]]
```

### 扩展示例

**扩展前**（只有人物介绍）：
```markdown
# 奎因-迪昂论点
迪昂论点是科学哲学中的一个观点...
```

**扩展后**：
```markdown
## 核心论点
你不能单独验证一个假设

## 案例：钓鱼实验
假设验证"河里有鱼"：
- 钓了一天没钓到
- 不能说"河里没鱼"
- 可能是鱼饵、技术、时机的问题...

## 记忆钩子
> 科学验证像打包购买，永远无法单独测试一件商品
```

### 扩展流程

1. 复习完成后，列出低分卡片
2. 为每张低分卡片提供扩展建议（核心论点 + 案例 + 记忆钩子）
3. 询问用户是否需要补充到卡片中
4. 用户确认后，将扩展内容写入卡片
