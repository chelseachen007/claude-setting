# 开发助手 - 工作流程

## 标准开发流程

### Step 1: 理解任务
- 明确任务类型：修 bug、加功能、重构、审查 PR、处理 Issue
- 确认目标仓库和分支
- 了解相关代码上下文

### Step 2: 环境准备
- 确认项目目录和技术栈
- 检查依赖是否安装完整
- 确认测试和构建命令

### Step 3: 代码分析
- 读取相关源码，理解现有逻辑
- 使用 project-analyzer 深度分析（大型任务）
- 标注需要修改的文件和影响范围

### Step 4: 实施改动
- 按最小改动原则修改代码
- 匹配项目现有代码风格
- 改动涉及多个文件时先列出文件清单

### Step 5: 验证
- 运行测试（test）
- 运行代码检查（lint）
- 运行构建（build）
- 确保三步全部通过

### Step 6: 提交
- 使用 conventional commit 格式
- 提交信息说「为什么」不说「做了什么」

## 特殊场景

### Issue 处理
1. 读取 Issue 描述，理解问题
2. 定位相关代码
3. 分析根因（不绕过）
4. 给出排查结论或修复方案
5. 用户确认后实施

### PR 审查
1. 读取 PR diff
2. 关注：逻辑正确性、安全性、性能、风格
3. 输出审查意见

### Bug 修复
1. 先用测试复现问题
2. 定位根因
3. 修复
4. 确保测试通过
