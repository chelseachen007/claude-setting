Hooks（生命周期钩子）
在 post-tool-use 或 pre-commit 等生命周期事件时自动运行 Shell 命令
Why for you: 你花了多个会话设置自动提交工作流、声音通知和备份脚本——Hooks 可以将这些形式化为可靠的自动化，在正确的时刻触发而无需手动调用。
// In .claude/settings.json { "hooks": { "PostToolUse": [{ "matcher": "Bash", "command": "afplay /System/Library/Sounds/Glass.aiff" }], "Stop": [{ "command": "cd /path/to/project && git add -A && git commit -m 'auto: session update' --allow-empty" }] } }
Headless 模式（无头模式）
从脚本和 cron 定时任务中非交互式运行 Claude
Why for you: 你已经有用于每日学习日志和备份的 cron 定时任务——Headless 模式让你可以自动化日志生成本身，而不仅仅是触发提醒，让 Claude 在无人值守的情况下分析会话并写笔记。
# In crontab, e.g., daily at 22:00 0 22 * * * cd ~/obsidian-vault && claude -p "Run the daily-learning-journal skill: analyze recent Claude Code sessions and write today's learning journal entry to the Obsidian vault." --allowedTools "Read,Write,Bash,Glob" 2>&1 >> ~/claude-journal.log

定义为 Markdown 的可复用提示，通过单个 /命令 运行
Why for you: 你已经安装了约 36+ 个技能并定期使用 /commit 和 /daily-learning-journal，但为你的重复模式（批量重构、微信账单解析、旅行指南提取）创建项目特定的技能可以避免每次重新解释上下文。

Apply change to ONE file first 3. Show the diff and ask for confirmation 4. If approved, apply to all matching files 5. Report summary of changes Always confirm the exact pattern on one file before batch applying.

批量模式重构
在扩展到 20+ 文件之前，始终在单个文件上验证模式。
你的会话显示多个大规模重构任务（约 20 个文件的 todoWork 草稿模式、otherParams 模式纠正、leasedProperty 模块）。摩擦模式是一致的：Claude 在许多文件中应用了略微错误的模式，需要在批量处理中途纠正。通过将"先在一个文件上验证"工作流制度化，你可以及早发现模式不匹配，并在 Vue/TypeScript 代码库中节省大量返工时间。
粘贴到 Claude Code：
I need to apply a refactoring pattern across multiple files. First, show me the exact change on ONE representative file. Wait for my confirmation before applying to all remaining files. After batch completion, show a summary of all changes made.

并行 Agent 批量重构
你的草稿模式 API 模式在 20 个文件中顺序应用——想象一下生成并行的 Claude Agent，每个同时处理一组文件，自动验证类型检查和 lint 规则，然后合并结果。整个代码库迁移可以在几分钟而不是几小时内完成，每个 Agent 运行自己的测试套件在合并前捕获回归。
开始方式： 使用 Claude Code 的 Agent 工具或 GitHub Actions 将多个 Headless Claude 进程分派到文件组，然后用合并脚本聚合结果。
粘贴到 Claude Code：
Analyze all files matching 'src/**/todoWork*' and 'src/**/list*.vue'. Group them into 4-5 clusters of related files. For each cluster, spawn a parallel sub-agent that: (1) applies the draft-mode API pattern with correct otherParams (version/mainId), (2) runs TypeScript type-checking on modified files, (3) reports any errors. After all agents complete, merge results and run the full test suite. If any agent fails, retry only that cluster with the error context included.
测试驱动自主 Bug 修复
你的会话显示出反复出现的错误方法（19 次）和有 Bug 的代码（7 次）模式，需要多轮纠正。一个自主循环——Claude 先写一个失败的测试，然后迭代修复直到测试通过——无需人工干预——将完全消除来回沟通。这可以一夜之间解决整个 Bug 积压。
开始方式： 将 Claude Code 与监听模式或 CI 触发器结合：指向失败的测试或错误日志，让它在有界重试预算内自主循环。
粘贴到 Claude Code：
Run the project's test suite and identify all failing tests. For each failure: (1) read the test file and the relevant source code, (2) write a minimal reproduction test if one doesn't exist, (3) propose a fix, (4) run the test again, (5) iterate up to 5 times until the test passes. Log each attempt's diff and test output. After all tests pass or max retries are hit, commit each successful fix as a separate commit with a descriptive message like 'fix: [description] (autonomous)'.
自主知识管道编排
你的每日学习日志、微信文章提取、视频转旅行指南工作流和技能管理都遵循一个目前手动触发的管道模式。想象一个定时编排器，自动醒来，发现各平台的新内容，分派专门的 Agent 进行提取/摘要/分类，优雅地处理失败并重试，将结构化输出写入 Obsidian——全部无需你打开终端。
开始方式： 利用现有的 cron 定时任务和 Claude Code Hooks 构建一个管道调度脚本，将技能与错误恢复链接在一起。
粘贴到 Claude Code：
Create a pipeline orchestration skill called 'daily-knowledge-pipeline' that: (1) checks configured RSS feeds, WeChat bookmarks, and Bilibili watchlists for new content since last run, (2) for each new item, spawns a sub-agent to extract content (using bb-browser for web, yt-dlp for video transcripts, OCR for images), (3) passes extracted content to a summarization agent that generates structured Obsidian notes with frontmatter, (4) runs a quality check agent that verifies notes have all expected fields, (5) commits new notes and updates the daily learning journal index. Store pipeline state in ~/.claude/pipeline-state.json so it resumes cleanly on failures. Schedule via cron at 6am daily.
