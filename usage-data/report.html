自主测试驱动功能开发

代理可以分析需求，先编写失败的测试，跨多个文件实现功能，迭代运行测试，并自我修正直到所有测试通过——无需人工调试周期即可交付生产就绪的代码。代理将自主处理完整的红绿重构循环，在呈现最终解决方案之前捕获边缘情况和类型错误。

我需要实现 [功能描述]。遵循这个自主工作流： 1. 首先，探索代码库以了解架构、模式和现有的类似功能 2. 编写全面的失败测试，覆盖正常流程、边界情况和错误处理 3. 跨所有必要文件实现功能（models、services、routes、UI 组件） 4. 循环运行测试，自己修复每个失败而不问我——迭代直到所有测试通过 5. 运行类型检查和 lint，修复所有问题 6. 只有在全部绿色时才汇报结果和总结 如果单个问题尝试 5 次后仍卡住，再报告阻塞点。
并行多代理代码库迁移

多个专门的代理可以同时处理大型迁移的不同模块——每个处理不同的领域（认证、数据层、UI 组件、API 路由），同时通过共享上下文进行协调。这将长达一周的迁移缩短到几小时，代理理解依赖关系、保持一致性并连贯地合并他们的工作。

我需要将此代码库从 [当前技术栈] 迁移到 [目标技术栈]。执行并行代理迁移策略： 1. 首先，分析整个代码库并创建依赖图，显示哪些模块可以安全地并行迁移 2. 启动 4 个并行子代理——每个分配到不同的领域（如代理1：认证和用户管理，代理2：数据库和模型，代理3：API 路由和业务逻辑，代理4：前端组件） 3. 每个代理应该：读取共享迁移模式、实现他们的领域、运行测试、自主修复问题 4. 所有代理完成后，运行完整的集成测试并修复任何跨模块问题 5. 提供统一的迁移报告 在检查点之间自主工作——只在关键决策点问我问题。
自愈式开发环境守护

持久的代理可以监控你的开发工作流，自动修复构建错误、解决依赖冲突、更新配置，甚至在运行时问题发生时进行调试。代理不是中断你的流程来修复环境问题，而是静默地维护健康的代码库——运行构建、捕获回归、在你遇到问题之前自我修复。

作为这个项目的自愈式开发环境守护。你的任务： 1. 运行完整的测试套件和类型检查——捕获所有当前的错误和警告 2. 对于每个问题，自主调查根本原因、实现修复并验证它解决了问题 3. 通过创建在文件更改时运行测试/类型检查并自动修复问题的脚本来设置持续监控 4. 修复你发现的任何依赖问题、配置冲突或环境设置问题 5. 初始清理后，提供健康报告：发现的问题、应用的修复、剩余的已知限制、以及会减少未来摩擦的架构改进 自主操作——在你安全可以修复的所有问题上无需确认。
