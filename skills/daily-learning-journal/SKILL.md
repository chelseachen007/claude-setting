# Daily Learning Journal & Weekly Review

自动总结每日的 Claude Code 会话记录，提取学习知识点和未完成事项，记录到 Obsidian 行思录中。**周五自动切换为周复盘模式。**

## 使用方法

在 Claude Code 中运行：
```
/daily-learning-journal
```

或者直接说：
- "总结今天的学习" / "写日记" / "今日复盘" → **日复盘**
- "周复盘" / "写周复盘" → **强制周复盘**
- 周五说"总结今天的学习" → **自动切换周复盘**

## 模式检测

```bash
DOW=$(date +%u)  # 5=周五
```

- **周五** → 自动进入周复盘模式
- **其他** → 日复盘模式
- 用户可手动指定模式覆盖自动检测

## 功能特点

### 日复盘

1. **自动分析今天的会话记录**，提取：
   - Skills 使用统计
   - MCP 工具调用统计
   - 创建/更新的文件
   - 技术主题、项目活动
   - 未完成事项
   - 关键洞察（3-5 条）
2. **🍞 今日面包屑**：记录学习路径（起点→探索→卡住→接续点）
3. **🤖 自动化发现扫描**：根据今日会话内容，识别可自动化的点（见下方「自动化发现检查」）
4. **按日期组织**：`行思录/年/年-月/日.md`
5. **智能更新**：已存在则追加，不存在则创建

### 周复盘

1. **面包屑路径串联**：把本周每天的学习路径串成完整轨迹
2. **目标回顾**：对照上周的「下周方向」
3. **知识产出统计**：新增卡片、wikilink 健康检查
4. **项目进展汇总**
5. **间隔复习状态**
6. **方法论反思与迭代**
7. **下周方向**：聚焦主题 + 最多 3 件行动 + 面包屑起点

## 配置

默认配置：
- 日记目录：`行思录/`
- 日期格式：`YYYY-MM-DD`
- 日复盘文件名：`YYYY-MM-DD.md`
- 周复盘文件名：`W-YYYY-MM-DD.md`
- 周复盘触发日：周五（`date +%u` = 5）

## 🤖 自动化发现检查（日复盘步骤之一）

在日复盘分析会话内容时，额外扫描以下四类信号。**只在发现明确可自动化的点时才记录**，不要生搬硬套。

### 四个维度与识别信号

| 维度 | 信号关键词 | 识别标准 |
|------|-----------|---------|
| **一：加速手动流程** | 复制粘贴、重复操作、手动转换、格式整理、逐个修改 | 同一操作做了 3 次以上 |
| **二：认知扩界** | 没想到、新方案、边界情况、隐藏依赖、替代思路 | AI 或他人提供了初始思路之外的有价值补充 |
| **三：解锁不经济任务** | 一直没做、以前太耗时、全量扫描、批量处理、人力不可能 | 一个有价值的任务因成本原因从未被执行，现在可行 |
| **四：专属工具** | 通用工具不行、和我们项目强耦合、定制脚本、个人工作流 | 解决方案高度特定于个人/项目，无法通用化 |

### 记录方式

1. 在日复盘文件中加一个 `## 🤖 自动化发现` 章节，列出当日发现的点（如有）
2. 同时追加到 Obsidian 库的 `private/关于自动化/AI四维实践记录.md`，按该文件中的记录模板格式写入

### 判断标准（不要过度记录）

- **必须记录**：当天手动做了、且有可能自动化但**还没有自动化**的方向。重点发现新的自动化机会
- **不记录**：已经在自动化的事情（如已有 Skill/脚本在跑的）、纯对话/探索、一次性任务
- **核心原则**：自动化发现 = 发现"可以但还没做的自动化机会"，不是记录已有的自动化成果
- 每天最多记录 3 条，宁缺毋滥
