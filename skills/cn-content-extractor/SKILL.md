# 中文平台内容提取

## 铁律

**绝不编造内容。** 必须从实际来源获取内容。如果所有提取方式都失败，立即告知用户让其手动粘贴，不要生成任何替代内容。

## 三层降级策略

按顺序尝试，任一层成功即停止，返回提取的内容。

### 第1层：直接抓取

用 `mcp__web_reader__webReader` 抓取 URL，获取 markdown 格式内容。

**成功判断**：返回的文本包含实质性正文内容（非空、非纯导航/广告、非登录提示、非验证码页面）。

**对以下平台通常有效**：掘金、简书、CSDN、36氪、少数派、豆瓣、知乎专栏（部分）

**失败标志**：
- 返回空内容或极短内容（<100字）
- 内容仅为"请在客户端打开"、"环境异常"、"验证码"等提示
- 返回的是登录/注册页面内容

### 第2层：浏览器自动化（web-access CDP）

第1层失败时，加载 `web-access` skill，使用 CDP 浏览器模式访问页面。

适用场景：
- 微信公众号文章（mp.weixin.qq.com）
- 需要登录态的内容（知乎、小红书）
- JS 动态渲染的页面
- 反爬机制较强的平台

**操作要点**：
1. 先 `check-deps.sh` 检查 CDP 可用性
2. 用 `/new` 打开目标 URL
3. 用 `/eval` 提取正文内容（优先提取 `article`、`.article-content`、`#content` 等语义容器）
4. 如遇"阅读全文"按钮，用 `/click` 点击展开
5. 提取完成后 `/close` 关闭 tab

### 第3层：缓存/镜像搜索

前两层都失败时，搜索缓存或镜像版本：

1. **Google 缓存**：用 WebSearch 搜索 `cache:{原始URL}`
2. **网页存档**：
   - 尝试 `https://webcache.googleusercontent.com/search?q=cache:{URL}`
   - 尝试 `https://web.archive.org/web/*/{URL}`
3. **内容聚合搜索**：用 WebSearch 搜索文章标题关键词，查找转载/镜像版本
4. **Jina Reader**：尝试 `https://r.jina.ai/{URL}` 作为最后的静态提取手段

### 全部失败

如果三层策略全部失败，**立即**明确告知用户：

> 无法从该 URL 提取内容（原因：[具体失败原因]）。请手动打开链接，复制正文内容粘贴给我，我会帮你处理。

## 平台特定策略

识别 URL 域名后应用对应策略：

| 平台 | 域名 | 推荐策略 | 备注 |
|------|------|---------|------|
| 微信公众号 | mp.weixin.qq.com | 直接跳到第2层 | 反爬严格，直接抓取几乎必然失败 |
| 知乎 | zhihu.com | 第1层 → 第2层 | 专栏文章第1层常有效，问答可能需要登录 |
| 百度百家号 | baijiahao.baidu.com | 第1层 → 第2层 | 内容在 JS 中渲染 |
| B站专栏 | bilibili.com/read | 第1层 | 通常直接抓取有效 |
| 掘金 | juejin.cn | 第1层 | 通常直接抓取有效 |
| 简书 | jianshu.com | 第1层 | 通常直接抓取有效 |
| CSDN | csdn.net | 第1层 | 可能需处理登录提示弹窗 |
| 豆瓣 | douban.com | 第1层 → 第2层 | 长评可能需登录 |
| 36氪 | 36kr.com | 第1层 | 通常直接抓取有效 |
| 少数派 | sspai.com | 第1层 | 通常直接抓取有效 |
| 小红书 | xiaohongshu.com | 第2层 | 反爬极严格，需 CDP |
| 微博 | weibo.com / m.weibo.cn | 第1层（移动版）→ 第2层 | 优先尝试 m.weibo.cn |

## 内容清洗

提取成功后进行清洗：

1. **去除噪声**：移除广告、推荐阅读、"扫码关注"、版权声明等非正文内容
2. **保留结构**：保持标题层级、列表、引用等 markdown 结构
3. **保留图片**：保留图片链接和 alt 文本
4. **保留元数据**：提取作者、发布时间、来源等信息，附在内容头部

## 输出格式

```markdown
# [文章标题]

> 来源：[平台名] | 作者：[作者] | 发布时间：[时间]
> 原始链接：[URL]

[正文内容]
```
